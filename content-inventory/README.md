# Content Inventory

**Purpose:** Organized repository of all learning content, structured hierarchically from high-level pillars down to individual lessons.

## Directory Structure

```
content-inventory/
├── schema/
│   └── schema.sql          # Database schema definition
├── sample-data/            # Example content for testing
├── pillars/                # Top-level content areas
│   └── {pillar-name}/
│       ├── pillar-{pillar-name}.md
│       └── learning-paths/
│           └── {path-name}/
│               ├── path-{path-name}.md
│               └── courses/
│                   └── {course-name}/
│                       ├── course-{course-name}.md
│                       └── modules/
│                           └── {module-name}/
│                               ├── module-{module-name}.md
│                               └── lessons/
│                                   └── lesson-{lesson-name}.md
```

## Content Hierarchy

1. **Pillar** - Top-level thematic area (e.g., "DataOps Engineering")
2. **Learning Path** - Structured learning journey within a pillar (e.g., "Apache Airflow Orchestration")
3. **Course** - Grouped modules covering specific skills (e.g., "Airflow DAG Development")
4. **Module** - Topic grouping within a course (e.g., "Task Dependencies")
5. **Lesson** - Individual learning unit (e.g., "Branching Logic with BranchPythonOperator")

## Database

Content metadata is tracked in SQLite database:
- **File:** `content.db` (generated by `scripts/init-database.ps1`)
- **Schema:** `schema/schema.sql`
- **Operations:** Use PowerShell scripts in `scripts/` directory

## Content Standards

All content files must:
- Include YAML frontmatter (see `.github/instructions/content.md`)
- Follow naming conventions (kebab-case)
- Use canonical tags from approved taxonomy
- Include prerequisites and learning objectives
