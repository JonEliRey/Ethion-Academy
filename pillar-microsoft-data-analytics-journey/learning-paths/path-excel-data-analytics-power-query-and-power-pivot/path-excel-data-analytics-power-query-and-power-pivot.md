# Excel Data Analytics: Power Query and Power Pivot

## Overview

Moving from manual Excel cleanup to automated analytics workflows is one of the most impactful productivity gains for business analysts. This learning path teaches you how to leverage Power Query for automated data ingestion and transformation, and Power Pivot for building reusable analytical models—all while staying in Excel.

**What You'll Build**: A production-ready Excel analytics workbook that ingests data from multiple sources, automates cleanup and transformations, builds a star schema data model, and delivers refreshed dashboards in under 5 minutes—all using the World Wide Importers sample database.

**Why This Matters**: Organizations waste countless hours on manual data preparation and spreadsheet maintenance. Power Query and Power Pivot skills are increasingly required for analyst roles, with 40%+ of business intelligence positions expecting these capabilities. More importantly, you'll transform 4-hour weekly reporting tasks into 15-minute refresh operations.

**Journey Outline**: Start with modern Excel setup and data mindset shifts → Master Power Query for data ingestion and transformation → Build analytical models with Power Pivot → Complete a capstone project demonstrating end-to-end automated analytics.

---

## Learning Objectives

By completing this learning path, you will be able to:

1. **Automate Data Ingestion with Power Query**
   - Connect to SQL databases, CSV files, and folder sources
   - Configure query parameters for environment flexibility
   - Implement data quality checks and profiling
   - Build reusable connection patterns

2. **Transform Data with Power Query**
   - Apply cleaning and shaping transformations
   - Create custom functions and reusable logic
   - Handle merges, lookups, and complex joins
   - Implement error handling and diagnostics

3. **Model Data with Power Pivot**
   - Design star schema relationships
   - Create measures and KPIs using DAX
   - Build role-playing dimensions
   - Optimize model performance

4. **Deliver Automated Analytics**
   - Configure refresh automation and scheduling
   - Build executive-ready pivot table dashboards
   - Implement conditional formatting tied to measures
   - Document and maintain analytics solutions

## Prerequisites

### Required Knowledge
- **Excel Fundamentals**: Confident navigation of workbooks, worksheets, cells, and basic Excel operations. You should be comfortable working in Excel and understand how data is organized.
- **Pivot Table Experience**: Ability to create basic pivot tables, add fields, and understand how they summarize data. This foundation is critical for Power Pivot concepts.
- **Data Concepts**: Understanding of tables, columns, rows, and basic data types (text, numbers, dates). Familiarity with how data is organized in databases or structured files.

### Recommended Preparation
- Review basic database concepts (tables, relationships) if completely new to relational data
- Practice creating pivot tables with sample data to ensure comfort
- Watch introduction to Power BI (optional) to understand broader Microsoft analytics ecosystem

### Tools & Environment
- **Microsoft Excel**: Version 2016 or later (Microsoft 365 recommended). Power Query is built-in; Power Pivot may require enabling.
- **Power Query Add-in**: Verify it's enabled via Data tab. Installation instructions provided in Module 1.
- **Power Pivot Add-in**: Must be enabled via Excel Options > Add-ins. Step-by-step guidance in Module 1.
- **World Wide Importers Database**: Microsoft sample database. Download options include SQL backup (requires SQL Server/LocalDB) or pre-extracted CSV files (no database required). Full setup guidance provided.

## Leader's Lens

> **For Decision Makers**: Understanding the business impact of this learning path

### Business Outcomes
- **Productivity Gains**: Reduce weekly reporting cycles from 4+ hours to 15-minute refresh operations, freeing analysts for higher-value analysis
- **Data Quality**: Automated transformations eliminate manual copy-paste errors and ensure consistent business logic across reports
- **Scalability**: Analytics solutions built with Power Query/Pivot scale from hundreds to millions of rows without redesign

### Decision Criteria
When evaluating this technology/approach, consider:
- **Excel Ubiquity**: Leverages existing Excel investments; no new tools to purchase or deploy
- **Migration Path**: Power Query and Power Pivot models import directly into Power BI Desktop, protecting investment when transitioning to enterprise BI
- **Governance Trade-offs**: Excel-based solutions need clear version control and refresh scheduling; consider Power BI Service for enterprise-scale refresh automation

### Risk Factors
- **Add-in Availability**: Power Pivot may require IT enablement in locked-down environments; address early in pilot
- **Performance Limits**: Excel-based models should stay under 100MB for responsiveness; larger datasets may require Power BI migration
- **Knowledge Transfer**: Document Power Query logic clearly; M code can become opaque without proper commenting

### Key Performance Indicators
- **Time Savings**: Measure hours saved per reporting cycle (target: 60-80% reduction)
- **Error Reduction**: Track data quality incidents before/after automation (target: 90% reduction in manual errors)
- **Adoption Rate**: Monitor how many analysts build automated solutions after training (target: 70%+ within 90 days)
